Understanding Architecture

Server and Client

Server Brief

The deep learning model will be in the Server. The API was made with flask. When the client sends a GET request to the server, the server accepts the input and returns the required intent or response. Instead of just intents for tasks, it also has intents for chat purposes. You can train your own model with your own data, and save the model in .h5 format as 'model.h5' in 'etc/model/saved'.

Also you need a valid API token to start sending requests to this server. The tokens can be generated by clicking the link below. You can perform a maximum of 200 requests per day. As this involes deep learning, hosting a model on the cloud can be expensive. You cannot register multiple tokens, your token will be attached with your base IP address. So more than 200 requests/day may result in a permanent Ban

Generate API Token

Client Brief

The client script will be used by users. You need to paste the API key in your 'client_config' script. Otherwise you wont be allowed to access the Server. Most of the tasks will be done in the client. For example, when the server returns
'--weather' flag, your client script will find your city name with your IP address, and will search for the weather info and return it to you. This was the reason why we didn't put these tasks for the server to do.

There will be other Assisting Scripts as well, such as the Speech Recognition or WakeWord Detection scripts if you are building a voice assistant.

USAGE: You just need to inherit Client class from 'client.py' to your class for production purposes, or just use the readymade 'client_wrapper.py' which has all the other scripts bound to it(Speech Recogniton, Text to Speech, and few other)

Understanding Intents

What are Intents?

There are few tags or inents assigned to our text data. The tags look something like this: '--task'. The Model has trained on these labels and text patterns. If your input get classified, then the server returns the predicted intent with the highest accuracy to you(your client script).

Weather Intent

When you send your query to the api, if something related to weather is asked, then it will return what intent is asked along with any response if present.
For example, if your QUERY was:

Can you tell me the weather?


then it will return a particular key(intent), here it will be "--weather", and your client handler script will be able to get the required info about weather by locating your city with your ip address.
NOTE: You have to register your OpenWeather API and paste it in the client config script to get weather search results.

Youtube/Music Intent

If your QUERY was:

Can you play jazz music on youtube?


then it will return a particular key(intent), here it will be "--youtube" along with your query, and your client handler script will parse youtube, download the song, and play it on a different process or thread.

Information Intent

If your QUERY was:

Do you know who is Donald Trump?


then it return a particular key(intent), here it will be "--info" along with your query, and your client handler script will parse the query to only "Donald Trump" and parse google or wikipedia for it, and return the information that it found.

News Intent

If your QUERY was:

Can you tell me the recent news?


then it return a particular key(intent), here it will be "--news", your client handler script might ask you the number of headlines you want, the data type will be of class int. Then it will parse google news and return a list of headlines, the first headline being the most recent.

Learn More

Notes Intent

If your QUERY was:

Can you write this down


then it return "--news" as intent flag, , your client handler script will then ask what to note, and you just need to dictate what to write, and it will be saved in .txt format. The client script is really flexible, you can also add smtplib to client to send the notes to your email.

Jokes Intent
